# CF ingress proxy worker

Authoritative app docs: `apps/cf-ingress-proxy-worker/README.md`.

## Safety rules

- Never hand-edit `apps/cf-ingress-proxy-worker/sql/*.ts`. They are generated by TypeSQL.
- If you edit any SQL source (`sql/*.sql`), regenerate wrappers with `pnpm -C apps/cf-ingress-proxy-worker run db:types`.
- If you edit schema (`schema.sql`), also add a new migration in `apps/cf-ingress-proxy-worker/migrations/`.
- Keep `schema.sql` and latest migration state aligned. If they diverge, TypeSQL output and deployed D1 schema can drift.
- Keep resolver ordering semantics intact unless intentionally changed:
  - exact pattern should beat wildcard
  - more specific wildcard should beat less specific wildcard
- Route create/update writes are admin-path operations. Proxy request latency depends on resolver query path, not these writes.

## Change checklist

Run from repo root:

```bash
pnpm -C apps/cf-ingress-proxy-worker run db:types
pnpm --filter @iterate-com/cf-ingress-proxy-worker typecheck
pnpm --filter @iterate-com/cf-ingress-proxy-worker test
```

For behavior changes in match ordering or proxy forwarding:

```bash
INGRESS_PROXY_E2E_BASE_URL=<https://...workers.dev> \
INGRESS_PROXY_E2E_API_TOKEN=<token> \
pnpm --filter @iterate-com/cf-ingress-proxy-worker test:e2e-live
```
