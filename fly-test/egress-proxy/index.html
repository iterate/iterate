  3. Here you should then see human-readable entries like Received proxied ... request and Upstream response ... for the same <http-...> ID.
  4. Sandbox blocked fetch should produce a line like Blocked by policy ... with an <http-...> ID.
  5. WS tests should show payload visibility and moderation. Your exact sent payload should appear in logs like Forwarding client message ..., Dropping client message ..., Holding client message for 30000ms ..., and Review wait complete ... sending held message onwards ....
